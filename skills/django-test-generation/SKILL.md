# Django Test Generation

Create **Django + DRF** tests that match the style in `backend/api/users/tests/` (notably `APITestCase`, `APIClient`, factories, explicit `expected_response` dicts, and role/permission matrices).

## Quick start (when skill is active)

1. Identify the code under test (view/action/serializer/helper) and the HTTP method + path.
2. Derive a small scenario set: happy path, auth/permissions, validation errors, not-found.
3. Write `APITestCase` tests using `APIClient`, project factories, and explicit assertions on:
   - status code
   - `response.json()` (either exact dict/list or key invariants)
   - database side effects

## Conventions to follow

### Test case structure

- Use `from rest_framework.test import APIClient, APITestCase` and `from rest_framework import status`.
- Prefer one feature-oriented class per file (or multiple related classes) named `Test<Area>` (examples in repo: `TestUserAuth`, `TestUserConfig`, `TestKellyConfig`).
- Use `setUp()` to create `self.client = APIClient()` plus factories for users/models you need.
- Use `self.client.force_authenticate(user=...)` to switch roles per scenario (unauthenticated vs authenticated, admin vs member vs non-member).

### Naming + docstrings

- Test methods: `test_<action>_<scenario>` (or `test_<action>` when the docstring is descriptive).
- Each test should have a docstring describing endpoint behavior, usually including method + path, e.g.:
  - `"""PATCH /api/users/auth/me/ with first_name and last_name only updates name."""`
  - `"""DELETE /api/users/kellys/:id/delete/ — admin deletes; member 403; non-member 404."""`

### Assertions (match existing style)

- Status codes: `self.assertEqual(response.status_code, status.HTTP_200_OK)` etc.
- JSON: `data = response.json()` then:
  - Prefer `expected_response = {...}` and `self.assertEqual(data, expected_response)` when stable.
  - When values are naturally variable (timestamps/serialized decimals/ids), assert invariants:
    - key presence (`self.assertIn("split_config", data)`)
    - value relations (e.g. `.endswith(".webp")`)
    - echoing request fields
    - timestamp formatting quirks (e.g. if comparing `isoformat()`, normalize `+00:00` to `Z` when needed)
- DB side effects:
  - Use `.refresh_from_db()` and assert updated fields.
  - Use `Model.objects.filter(...).exists()` for create/delete/membership side effects.

### Organization for multi-endpoint suites

When a file covers multiple endpoints/actions, group tests with section banners:

```python
# -------------------------------------------------------------------------
# Create
# -------------------------------------------------------------------------
```

## Workflow (how to generate tests from code)

1. **Locate entry point**: view/action function, serializer validation, helper, or backend.
2. **Extract behaviors**:
   - inputs (payload fields, path params)
   - permissions/roles
   - validation errors (serializer fields, model constraints)
   - side effects (creates/updates/deletes/blacklists/joins/leaves)
3. **Choose scenarios** (keep small but high-signal):
   - happy path
   - unauthenticated (401) if endpoint is protected
   - forbidden role (403) when applicable
   - not found / no access (404) when applicable
   - invalid payload (400) for the most important validations
4. **Implement**:
   - Arrange with factories
   - Act with `self.client.<method>(path, payload, format="json")` (or `format="multipart"`)
   - Assert status, JSON, and DB changes

## Templates (adapt, don’t copy blindly)

### APITestCase + endpoint test

```python
from rest_framework import status
from rest_framework.test import APIClient, APITestCase

from api.users.factories import UserFactory  # adjust to feature factories


class TestSomeFeature(APITestCase):
    """Test cases for <feature/endpoint group>."""

    def setUp(self):
        self.client = APIClient()
        self.user = UserFactory()

    def test_some_behavior(self):
        """POST /api/.../ — happy path returns 201 and creates object."""
        self.client.force_authenticate(user=self.user)
        payload = {"field": "value"}

        response = self.client.post("/api/.../", payload, format="json")

        self.assertEqual(response.status_code, status.HTTP_201_CREATED)
        data = response.json()
        self.assertIn("id", data)
```

### Exact expected_response

```python
data = response.json()
expected_response = {
    "id": data["id"],  # if generated by API
    "name": payload["name"],
}
self.assertEqual(data, expected_response)
```

### DB side effect checks

```python
self.assertTrue(MyModel.objects.filter(pk=data["id"]).exists())
obj.refresh_from_db()
self.assertEqual(obj.some_field, "expected")
```
